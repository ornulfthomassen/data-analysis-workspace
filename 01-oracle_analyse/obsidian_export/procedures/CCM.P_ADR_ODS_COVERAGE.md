# P_ADR_ODS_COVERAGE

**Schema:** `CCM` | **Type:** `Procedure`

## Description
This procedure implements a "blue-green deployment" or rolling update strategy for the `ODS_COVERAGE` table. It populates a new temporary table (`ODS_COVERAGE_N`) with fresh data derived from various source tables, creates indexes on this temporary table, and then atomically swaps it with the main `ODS_COVERAGE` table. The old `ODS_COVERAGE` table is either dropped or renamed to a backup table (`ODS_COVERAGE_O`). The swap is conditional, based on percentage change in row counts, and includes logging of status and errors to `CLM_CCM.ODS_PROCEDURE_SWAP_STATUS`.

# P_ADM_CUSTOMER_USER_SUBS_AGG

**Schema:** `CLM_ADM` | **Type:** `Procedure`

## Description
This procedure aggregates customer user subscription and traffic data for a specified month (P_YYYYMM). It first validates the existence of source data. Then, it dynamically creates or reuses a partition in the target table 'ADM_CUSTOMER_USER_SUBS_AGG' for the given month. It populates a temporary table ('TMP_CUSTOMER_USER_SUBS_AGG') with aggregated data by joining various subscription, customer, product, and usage history tables. Finally, it uses an 'EXCHANGE PARTITION' operation to quickly swap the temporary table with the corresponding partition in the permanent 'ADM_CUSTOMER_USER_SUBS_AGG' table, effectively loading the aggregated data.

# P_ADM_FB_ACCESSES_CUST_HIST

**Schema:** `CLM_ADM` | **Type:** `Procedure`

## Description
This procedure populates or refreshes a monthly partition in the `ADM_FB_ACCESSES_CUST_HIST` table for a specified `YYYYMM` period. It first validates source data availability, then creates a new partition if it doesn't exist (or overwrites an empty one if allowed). Data is generated by joining and aggregating customer and product information from several source systems into a temporary table, which is then efficiently moved into the target table's partition using an `EXCHANGE PARTITION` operation. Finally, it rebuilds local indexes and collects statistics for the new partition.

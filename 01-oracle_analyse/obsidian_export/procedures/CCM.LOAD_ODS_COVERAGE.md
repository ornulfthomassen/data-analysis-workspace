# LOAD_ODS_COVERAGE

**Schema:** `CCM` | **Type:** `Procedure`

## Description
This procedure orchestrates the refresh of the ODS_COVERAGE table using a 'hot-swap' strategy. It first populates a temporary table (ODS_COVERAGE_TMP) with transformed data from various source systems. It then conditionally renames tables to promote ODS_COVERAGE_TMP to ODS_COVERAGE, optionally backing up the existing ODS_COVERAGE to ODS_COVERAGE_BKP, ensuring continuous availability of data during the update. The process includes schema creation, data loading, index creation, and statistics gathering, and logs the swap status.

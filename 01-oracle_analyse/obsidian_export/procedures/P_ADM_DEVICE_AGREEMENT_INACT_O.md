# P_ADM_DEVICE_AGREEMENT_INACT_O

**Schema:** `CLM_ADM` | **Type:** `Procedure`

## Description
This procedure extracts and processes detailed information related to device agreements, subscriptions, customer ownership, and IMEI usage for a specified month (`P_YYYYMM`). It consolidates data from various source tables into a temporary staging table (`TMP_DEVICE_AGREEMENT_INACTIVE`), then uses an exchange partition operation to load this data into a specific monthly partition of the permanent `ADM_DEVICE_AGREEMENT_INACTIVE` table. The primary goal is to maintain a historical snapshot of device agreement statuses, particularly focusing on inactive agreements, along with related subscription and device usage metadata.

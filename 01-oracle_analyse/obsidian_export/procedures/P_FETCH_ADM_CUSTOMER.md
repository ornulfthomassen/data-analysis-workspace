# P_FETCH_ADM_CUSTOMER

**Schema:** `CLM_ADM` | **Type:** `Procedure`

## Description
This procedure extracts customer administration data for the current month and conditionally for the previous month into separate, dynamically named, permanent tables. It first determines the current and previous month keys. It then checks for the existence of the respective target tables (e.g., ADM_CUSTOMER_YYYYMM), drops them if found, and subsequently creates new tables by selecting a comprehensive set of customer attributes from either 'VYA_ADM_CUSTOMER' (for previous month) or 'VYA_ADM_CUSTOMER_CURRENT' (for current month). The previous month's data loading is executed only if the current day of the month is between the 7th and 11th. The procedure includes logging of table creation and row counts, and raises an application error if a target table is not created or is empty.
